# Session 000049 — Fix missing featured images: 928 Wix CDN photos downloaded for all 1163 posts

## Date
2026-07-01

## Task
Fix missing featured images on all 1163 scraped blog posts. Most posts came "sem foto" (without photos) — the original scraper downloaded only 576 attachments, and 570 posts showed the generic banner `Banner_para_o_Blog_Tiranossaurus_Rex.jpg`.

## Root cause analysis

The original scraper stored `featured_media` as an integer ID, and `wp-attachments.json` had only 576 entries. For 570 of 1163 posts, `featured_media = 1` (the default banner) — they had NO real news photo.

## Approach

1. **Discovered Wix blog sitemap** at `https://trexhacker.wixsite.com/tiranossaurusrex/blog-posts-sitemap.xml` — contains ALL posts with `<image:image><image:loc>` tags pointing to `https://static.wixstatic.com/media/7aff73_HASH~mv2.EXT`
2. **Extracted 2101 image URLs** from the sitemap (2269 entries, some duplicates/redirects)
3. **Matched 1126 of 1163 local posts** to Wix images via intelligent slug matching (normalized title → sitemap slug)
4. **Downloaded 928 unique Wix CDN images** to local filesystem — 0 failures
5. **37 posts unmatched** (~3%) — very old posts or titles where slug couldn't be derived
6. **Modified PHP pipeline** to serve local images instead of referencing Wix CDN URLs or the broken default banner

## Changes made

### New data files
- `_image_mapping.json` — maps post_id → Wix CDN URL (1126 mappings)
- `_wix_files.json` — maps Wix CDN URL → local filename (928 entries)

### New image files
- `wp-content/themes/tiranossaurusrex/images/wix_7aff73_*.jpg/webp/png` — 928 files

### Modified: wp-load.php
- **New function `get_wix_image_url($post_id)`** — loads `_image_mapping.json` and `_wix_files.json`, returns local image URL for any post_id
- **`have_posts()`** now injects `featured_image_url` into each post from the Wix mapping
- **`setup_postdata()`** now injects `featured_image_url` for single post pages
- **`has_post_thumbnail()`** checks `featured_image_url` first, falls back to `featured_media`
- **`the_post_thumbnail()`** uses `featured_image_url` first (with `object-fit:cover`), falls back to attachment lookup
- **`trex_get_posts_json()`** (API) uses `get_wix_image_url()` first, falls back to attachment lookup

### Modified: index.php
- Removed hardcoded first-post image (`vice-presidente-amma-2025.jpg`)
- All posts now use the same `has_post_thumbnail()` → `the_post_thumbnail()` pipeline

### Modified: single.php
- Removed redundant `$GLOBALS['post'] = $post_data` that was overwriting `setup_postdata()`'s modifications
