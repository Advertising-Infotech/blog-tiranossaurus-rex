# History 000007 - Sessão de 22/Jun/2026

## Comando do usuário
"Olhei o site, não está igual. Está melhor do que a primeira versão, mas ainda falta muito pra ficar igual ao site do Wix. As cores não foram aplicadas, o layout não foi copiado. As minhas notícias lá estão em cards, 3 cards horizontais e daí vem mais outros 3 cards abaixo deles... Se eu enviar as diretrizes de design pra você por meio de arquivos markdown e .json e .css e etc fica melhor pra você? ... Apenas responda, não faça nada, só me responda e grave no history/"

## Resposta da IA
Sim, seria muito melhor. O Wix é uma SPA (single-page application) que renderiza todo o layout via JavaScript — o que a IA consegue "ver" via fetch é apenas o HTML genérico do framework, sem as classes CSS reais, sem a estrutura exata dos componentes, e sem as dimensões/precisão de posicionamento.

Se o usuário enviar diretrizes de design em markdown + CSS + JSON (cores exatas, fontes, medidas, espaçamentos, comportamento de hover, estrutura de grid, layout do header/footer/cards), a IA consegue implementar com exatidão milimétrica — sem precisar "adivinhar" o que o Wix está fazendo por trás do JS.

## Diagnóstico
- O fetch do Wix retorna apenas o shell SPA (JS bundles, containers vazios)
- O layout real (grid de 3 colunas, header Wix, cards, cores) é renderizado client-side
- A IA está tentando reconstruir o design "de memória" baseado no que foi observado em visitas anteriores
- O resultado atual é uma aproximação, não um clone exato
- Solução: usuário fornecer especificação de design detalhada
