### 3. Bug critical fixado: loop infinito
**Causa raiz**: `have_posts()` resetava o ponteiro do array a cada chamada, então o padrão WordPress `if (have_posts()) : while (have_posts())` causava loop infinito — o `while` sempre resetava para o primeiro post.

**Solução**: `have_posts()` só carrega/cacheia o array uma vez (primeira chamada). Usa contador `$wp_current_post` para trackear posição atual. `the_post()` lê por índice e incrementa contador.
