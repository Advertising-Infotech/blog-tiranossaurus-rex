# Session 000044 — Fixes: mobile title truncation + desktop nav bar width

## Date
2026-06-28 ~05:55

## Task
1. Mobile: "Tiranossaurus Rex" truncated to "Tiranossaur..." due to `white-space: nowrap`
2. Desktop: navigation bar background wider than header and content area

## Root causes
### Fix 1 — Mobile title
`h1` in header had `white-space: nowrap; overflow: hidden; text-overflow: ellipsis`. On small screens (<480px), the title "Tiranossaurus Rex" (~18 chars at 1.34rem) exceeded available width (~167px after profile photo, logo, gaps, and padding), causing truncation with ellipsis.

### Fix 2 — Desktop nav bar
`.main-navigation` background/borders filled 100% of `.site` (max-width: 1200px), while `.header-wrapper` has `padding: 0 30px` constraining `.site-header` gradient to 1140px. The nav background extended 30px beyond the header gradient on each side, making it appear visually wider.

## Changes made
### Fix 1 — Mobile (480px breakpoint)
- Added `white-space: normal` to `.header-text-block h1` in the @media (max-width: 480px) block
- Allows the title to wrap to two lines on small screens instead of truncating

### Fix 2 — Desktop nav
- Added `max-width: 1140px; margin: 0 auto;` to `.main-navigation` — constrains the nav background to match `.site-header` gradient width
- Removed `padding: 0 30px` from `.main-navigation .menu` — since the nav itself is now 1140px and centered (30px from each edge of .site), menu items start at the correct position without extra padding
- Mobile 768px rule preserves `padding: 0 15px` for the open menu
